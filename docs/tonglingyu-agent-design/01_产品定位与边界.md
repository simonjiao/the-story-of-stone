# 01 产品定位与边界

## 产品定位

本产品“通灵玉”是一个面向《红楼梦》的研究型对话助手，不是角色扮演陪聊机器人，也不是无证据的文学闲聊模型。

第一版目标是：

> 用户提出《红楼梦》相关问题后，系统能够检索原文、脂批、版本信息和人物事件资料，生成区分证据层级的回答，并经过审校后返回。

## 产品一句话

一块能查原文、引脂批、讲版本、辨推断的“通灵之玉”。

## 第一版核心能力

| 能力 | 说明 |
|---|---|
| 原文定位 | 能回答人物、事件、诗词、判词、回目等原文定位问题 |
| 脂批解释 | 能引用和解释脂批，但不把脂批混同为正文事实 |
| 版本辨析 | 能区分通行本、脂评本、前八十回、后四十回 |
| 人物事件梳理 | 能围绕核心人物梳理关系、经历和命运线索 |
| 证据分层 | 能区分正文可证、脂批提示、版本差异、合理推断、证据不足 |
| 审校拦截 | 能拦截无证据断言、版本混淆和过度推断 |

## 第一版不做什么

| 不做项 | 原因 |
|---|---|
| 不做林黛玉、贾宝玉等人格陪聊 | 容易偏离研究型目标 |
| 不做全量红学论文库 | 第一版先验证原文、脂批和证据链 |
| 不做复杂版本校勘系统 | 校勘工作量大，适合后续阶段 |
| 不做自由续写 | 容易混淆原著、续书和模型创作 |
| 不把电视剧、影视改编当作原著证据 | 保持文本研究边界 |
| 不暴露内部 Agent 给普通用户 | 避免用户绕过固定审校流程 |

## 用户角色

| 用户角色 | 主要诉求 |
|---|---|
| 普通读者 | 想知道情节、人物关系、诗词判词含义 |
| 深度读者 | 想区分原文、脂批、版本分歧和研究推断 |
| 红学爱好者 | 想查脂批、人物命运伏线和文本证据 |
| 教学/内容创作者 | 想获得可引用、可核验、可解释的回答 |
| 系统管理员 | 关注权限、审计、稳定性和可维护性 |

## 成功标准

第一版成功不看回答是否“文采好”，而看：

1. 是否能引用可追溯证据；
2. 是否能区分正文、脂批、版本和推断；
3. 是否能在证据不足时说“不足”；
4. 是否能通过审校 Agent 拦截不可靠回答；
5. 是否能在 Open WebUI 中只暴露一个清晰入口。

## 产品边界冻结条件

产品边界可以冻结的条件：

1. 第一版核心能力不再继续扩张；
2. 不做项已被业务方接受；
3. 回答证据分层方式已被接受；
4. 用户只看到一个“通灵玉”的入口已被接受；
5. Gateway 不是第 5 个 Agent 的原则已被接受。
