# 10 内部接口契约

## 目标

内部接口用于 Gateway、Hermes Agent、知识服务、证据服务和审校服务之间协作。本文档只定义语义契约，不绑定具体实现方式。

## 内部接口分组

| 接口组 | 调用方 | 被调用方 | 目的 |
|---|---|---|---|
| Agent 调用接口 | Gateway | Hermes Runtime | 调用指定 profile 执行任务 |
| 上下文裁剪接口 | Gateway | 上下文服务 | 为不同 Agent 生成可见上下文 |
| 正文检索接口 | text Agent 或工具层 | 知识服务 | 获取正文证据 |
| 脂批检索接口 | commentary Agent 或工具层 | 知识服务 | 获取脂批证据 |
| 版本比较接口 | commentary Agent 或工具层 | 知识服务 | 获取版本差异 |
| 人物事件查询接口 | main Agent 或工具层 | 知识服务 | 获取人物关系和事件链 |
| 证据包接口 | Gateway | 证据服务 | 创建、读取、更新证据包 |
| 审校接口 | Gateway | reviewer Agent | 审查草稿 |
| 审计记录接口 | Gateway | 审计服务 | 写入追踪和调用日志 |

## Agent 调用接口

### 目的

让 Gateway 以受控方式调用某个 Hermes profile 执行指定任务。

### 输入语义

1. 追踪编号；
2. 会话编号；
3. 目标 profile；
4. 任务名称；
5. 任务输入；
6. 工具权限约束；
7. 超时和重试策略；
8. 输出格式要求。

### 输出语义

1. 任务状态；
2. 结构化结果；
3. 调用摘要；
4. 错误类别；
5. 运行时长；
6. 是否符合 schema。

### 验收标准

1. 用户不能直接构造该接口请求；
2. profile 只能执行被授权任务；
3. 输出不符合 schema 时必须拦截；
4. 超时必须被 Gateway 处理。

## 上下文裁剪接口

### 目的

为不同 Agent 生成不同可见上下文，避免信息过载和权限混乱。

### 输入语义

1. 用户问题；
2. 会话摘要；
3. Agent 类型；
4. 任务类型；
5. 策略约束。

### 输出语义

1. 该 Agent 可见的问题内容；
2. 必要的上下文摘要；
3. 禁止使用的上下文范围；
4. 可调用工具列表；
5. 输出约束。

### 验收标准

1. text Agent 不获得完整用户历史；
2. commentary Agent 不获得无关正文库原始数据；
3. reviewer 只获得草稿和证据包；
4. main 获得必要上下文但不直接获得数据库权限。

## 正文检索接口

### 输入语义

1. 检索问题；
2. 人物、回目、版本、段落类型等条件；
3. 返回数量；
4. 是否需要原文片段；
5. 是否需要支持范围和限制说明。

### 输出语义

1. 正文证据卡片；
2. 回目和位置；
3. 原文片段或摘要；
4. 支持范围；
5. 不支持范围；
6. 置信度。

### 验收标准

1. 必须返回来源；
2. 不返回无来源概括；
3. 不混入脂批；
4. 未命中时返回证据不足。

## 脂批检索接口

### 输入语义

1. 检索问题；
2. 版本条件；
3. 回目或人物条件；
4. 批语类型；
5. 是否需要对应正文。

### 输出语义

1. 脂批证据卡片；
2. 批语版本；
3. 对应正文位置；
4. 批语类型；
5. 支持范围；
6. 不支持范围；
7. 置信度。

### 验收标准

1. 必须标注版本；
2. 必须说明对应位置；
3. 不得将脂批作为正文事实返回；
4. 无法定位来源时标记证据不足。

## 证据包接口

### 目的

将证据卡片、结论声明和审校记录统一管理。

### 输入语义

1. 用户问题；
2. 证据卡片；
3. 结论声明；
4. 追踪编号；
5. 生成阶段。

### 输出语义

1. 证据包编号；
2. 证据卡片列表；
3. 结论声明列表；
4. 证据等级；
5. 审校状态。

### 验收标准

1. 每个复杂回答都有证据包；
2. 每条关键结论能映射证据；
3. 证据包可追溯到请求；
4. 证据包不可被普通用户任意篡改。

## 审校接口

### 输入语义

1. 用户问题；
2. 草稿答案；
3. 证据包；
4. 回答策略；
5. 负面清单。

### 输出语义

1. 是否通过；
2. 问题列表；
3. 严重程度；
4. 必须修订项；
5. 可保留结论；
6. 审校摘要。

### 验收标准

1. 无证据断言必须被拦截；
2. 脂批和正文混淆必须被拦截；
3. 前八十回和后四十回混淆必须被拦截；
4. 版本缺失必须至少警告；
5. 用户不能关闭审校。
