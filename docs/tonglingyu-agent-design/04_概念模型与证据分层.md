# 04 概念模型与证据分层

## 核心概念

| 概念 | 定义 |
|---|---|
| 用户问题 | 用户通过 Open WebUI 输入的自然语言问题 |
| 可见模型 | Open WebUI 中展示给用户的“通灵玉” |
| 内部 Agent | Runtime 内部运行的四个受控 profile |
| Gateway | OpenAI-compatible 入口、Runtime step plan、安全和审计层 |
| 证据卡片 | 一条可追溯的正文、脂批、版本或人物事件证据 |
| 证据包 | 一次回答所依赖的证据卡片集合 |
| 结论声明 | 最终回答中的一个可验证判断 |
| 审校意见 | reviewer 对草稿的通过、警告或拦截意见 |

## 证据类型

| 类型 | 说明 |
|---|---|
| 正文证据 | 来自《红楼梦》正文的回目、段落、诗词、判词、曲文 |
| 脂批证据 | 来自脂评系统的批语和对应正文位置 |
| 版本证据 | 来自不同版本之间的差异、缺失、增补或来源说明 |
| 人物事件证据 | 来自人物、关系、事件链和专题标签 |
| 字形读音证据 | 来自原文、批语、注释或人工校订的生僻字字形和读音说明 |
| 推断证据 | 基于多条证据形成的谨慎解释，不等同于文本事实 |

## 证据等级

| 等级 | 含义 | 表达方式 |
|---|---|---|
| A | 正文直接证明 | 正文明确写到 |
| B | 正文多处互证 | 综合多处文本可判断 |
| C | 脂批明确提示 | 脂批提供这种理解方向 |
| D | 合理推断 | 可以推测，但不能作为定论 |
| E | 证据不足 | 当前证据不足以证明 |

## 证据卡片字段

证据卡片至少包含：

| 字段 | 说明 |
|---|---|
| 证据编号 | 证据唯一标识 |
| 证据类型 | 正文、脂批、版本、人物事件、推断 |
| 来源版本 | 通行本、庚辰本、甲戌本等 |
| 回目或位置 | 证据所在位置 |
| 原文片段或摘要 | 可供引用或核验的内容 |
| 字形读音标注 | 保留生僻字、异体字、旧字形及其可追溯读音 |
| 支持范围 | 该证据能够支持的结论 |
| 不支持范围 | 该证据不能证明的结论范围 |
| 置信度 | 高、中、低 |
| 来源说明 | 说明来源、整理方式或局限 |

## 证据包字段

证据包至少包含：

| 字段 | 说明 |
|---|---|
| 证据包编号 | 一次回答的证据集合标识 |
| 用户问题 | 当前问题 |
| 证据卡片列表 | 所有参与回答的证据 |
| 结论声明列表 | 回答中产生的关键判断 |
| 证据等级 | 每个结论对应的证据等级 |
| 审校状态 | 通过、警告、拦截 |
| 生成时间 | 用于追溯 |
| 关联会话 | 关联 Open WebUI 会话和 Hermes 会话 |

## 结论声明要求

最终回答中所有关键判断都应能映射到至少一条证据卡片，尤其是：

1. 人物命运；
2. 人物关系；
3. 版本差异；
4. 脂批解释；
5. 作者意图类表述；
6. 对后四十回与前八十回关系的判断。

## 不允许的证据用法

1. 用脂批证明正文已经明写；
2. 用后四十回证明前八十回必然如此；
3. 用影视剧设定证明原著；
4. 用网络流行说法替代文本证据；
5. 用模型记忆替代检索证据；
6. 用单条模糊证据得出确定结论。

## 知识库对象

除证据卡片和证据包外，知识库层需要明确以下对象：

| 概念 | 定义 |
|---|---|
| 原文段落 | 带版本、回目、位置和段落类型的正文单位 |
| 规范化文本 | 用于检索的繁简、异体、标点等处理结果，不作为引用证据 |
| 批语单元 | 脂批、眉批、夹批、回前批、回末批等批注单位 |
| 版本记录 | 版本名称、系统、存回范围、来源状态和使用限制 |
| 人物别名 | 标准人物与称谓、别名、号、俗称之间的映射 |
| 关系记录 | 人物关系及其证据等级、版本范围和来源 |
| 事件节点 | 带原文证据的情节或人物经历节点 |
| 诗词判词单元 | 诗、词、曲、判词、灯谜、偈语等专题文本 |
| 字形读音标注 | 生僻字、异体字、旧字形及可追溯读音 |
| 不支持范围 | 某条证据不能证明的结论范围 |

“不支持范围”是证据卡片的必填边界。例如，某条脂批可以提示人物命运方向，但不能证明前八十回正文已经明写该结局。
