# 08 知识库与 RAG 设计

## 目标

知识库与 RAG 层负责为 Agent 提供可追溯证据，而不是直接生成最终回答。

## 知识库组成

| 知识库 | 用途 |
|---|---|
| 原文知识库 | 支撑正文定位、诗词判词、情节和人物事件 |
| 脂批知识库 | 支撑脂批解释、伏笔提示、版本批语 |
| 人物事件库 | 支撑人物关系、事件链、别名和命运线索 |
| 版本说明库 | 支撑不同版本、前八十回与后四十回的边界说明 |
| 字形读音索引 | 支撑生僻字、异体字、旧字形和读音标注的保真展示与检索 |
| 评测题库 | 支撑检索与回答验证 |

## 原文知识库

原文库记录：

1. 作品名；
2. 版本；
3. 回目；
4. 段落位置；
5. 段落类型；
6. 原文片段；
7. 涉及人物；
8. 涉及事件；
9. 关键词；
10. 生僻字、异体字或旧字形的字形与读音标注；
11. 来源说明。

段落类型包括：

1. 正文叙述；
2. 对话；
3. 诗；
4. 词；
5. 判词；
6. 曲文；
7. 回目；
8. 图画说明。

## 脂批知识库

脂批库记录：

1. 批语版本；
2. 对应回目；
3. 对应正文位置；
4. 批语原文；
5. 批语类型；
6. 涉及人物；
7. 支持范围；
8. 不支持范围；
9. 生僻字、异体字或旧字形的字形与读音标注；
10. 来源说明。

批语类型包括：

1. 伏笔；
2. 结局提示；
3. 人物评语；
4. 结构提示；
5. 作者意图；
6. 版本校勘；
7. 无法分类。

## 人物事件库

人物事件库记录：

1. 人物名称和别名；
2. 所属关系；
3. 首次出现位置；
4. 关键事件；
5. 与其他人物的关系；
6. 判词、曲文和伏线；
7. 证据来源。

第一版只做核心人物，不做全量人物。

## 生僻字与字形读音保留

资料处理和知识库构造必须保留生僻字、异体字、旧字形及其可追溯读音，不得在入库前把它们静默替换为通用字、简化字或拼音占位。

source snapshot、建库中间结果和最终证据卡片应遵循：

1. 原始字形按来源文本保留，检索规范化只能作为附加索引字段；
2. 来源中已经出现的 ruby、反切、拼音、注音或人工校订读音，应登记为 `rare_char_annotations`；
3. `rare_char_annotations` 至少包含 `glyph`、`pronunciation`、`source` 和来源位置；
4. 证据卡片展示时优先显示原始字形，并在需要时附带读音；
5. 来源没有读音时，只标记读音未知，不由模型补写为事实。

## 检索原则

RAG 层应使用混合检索思想：

1. 精确检索用于人名、回目、诗句、关键词；
2. 语义检索用于解释性问题；
3. 重排序用于减少相似但错误的证据；
4. 证据合并用于构造证据包；
5. 结果必须带来源和置信度。

本文档不绑定具体检索引擎、向量库或排序模型。

## 检索返回原则

RAG 不返回最终答案，只返回证据卡片。

证据卡片必须说明：

1. 证据是什么；
2. 来自哪里；
3. 能支持什么；
4. 不能支持什么；
5. 置信度如何；
6. 是否需要进一步审校。

## 验证指标

| 指标 | 说明 |
|---|---|
| 正文定位命中率 | 原文问题能否在前若干条命中正确回目 |
| 判词诗词命中率 | 判词、曲文、诗词是否准确定位 |
| 脂批来源准确率 | 脂批是否带版本和对应位置 |
| 人物关系可追溯率 | 人物关系是否能回溯到证据 |
| 证据误召回率 | 返回相似但无关证据的比例 |
| 证据不足识别率 | 无法证明时是否不强答 |

## 第一版建库范围

建议第一版范围：

1. 维基文库《红楼梦》全本正文；
2. 维基文库或同等可追溯来源中的前 80 回脂评相关正文；
3. 维基文库或同等可追溯来源中的关键脂批；
4. 第五回判词、曲文、册页图画；
5. 约 30 个核心人物；
6. 核心人物事件链；
7. 100 到 200 条评测问题。

## 第一版资料来源

第一版基础资料不使用旧的 `resources/base/hongloumeng/` 抽取产物。所有资料先进入 `resources/sources/` 下的 source snapshot，再由建库流程登记为具体证据类型。

优先来源：

1. 维基文库《红楼梦》全本，登记为 `base_material`。
2. 维基文库脂批本、脂评相关页面或其他可追溯公开来源，登记为 `commentary_material` 或 `extended_base_material`。
3. 人工整理并可追溯到正文/脂批的核心人物事件表，登记为 `extended_base_material`。
4. 标准问题集和负面用例，登记为 `evaluation_material`。

## 不做范围

第一版不做：

1. 全量红学论文库；
2. 全版本深度校勘；
3. 自动生成未经校验的人物关系；
4. 将所有网络资料直接入库；
5. 将模型生成内容反写为知识库事实。
