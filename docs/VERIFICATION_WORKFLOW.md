# 转录校订流程

转录文本只能作为初稿。校订的目标是把视频讲解、原文引用、脂批、章回名和术语区分清楚，并给关键修改留下可追溯证据。

## 标准步骤

1. 生成或更新转录。
   - 输出必须包含 `.txt`、`.srt`、`.transcript.json`。
   - 不急着全局替换同音词。

2. 术语初筛。
   - 查人名、章回名、回目、菜名、器物名、脂批常见表达。
   - 重点查 `官中/公中/宫中`、`分例/份例`、`宝黛钗`、`钗黛` 等高风险词。

3. 识别需要交叉验证的位置。
   - 出现“原著”“原文”“书中”“脂批”“写道”“第几回”等提示时，必须查基础版本文本。
   - 人名、菜名、器物名密集的段落，也要按章节查证。

4. 跨章节查证。
   - 不只查视频标题或开头提到的章节。
   - 先按转录中出现的章节线索查，再按关键词和语义相关段落扩展。

5. 区分文本类型。
   - 原文引用按基础版本文本。
   - 脂批、回目、注释按对应来源。
   - 讲解者口述不要强行改成原文句式。
   - `不红君` 是讲解者自称，应保留。

6. 同步三份输出。
   - 每次改动后，确认 `.txt`、`.srt`、`.transcript.json` 文本内容、顺序和数量一致。

7. 提交前复核。
   - 运行 JSON 格式检查。
   - 运行三文件一致性检查。
   - `git diff --stat` 确认没有音频、视频或临时文件。

## 第一条视频示例

第一条视频标题围绕第六十一回，但实际至少牵涉：

- 第五十一回：园中厨房、分例、总管房。
- 第六十回：茉莉粉、玫瑰露、芳官、夏婆子、小蝉等前置矛盾。
- 第六十一回：莲花儿点鸡蛋、司棋打砸厨房、柳家的和小厮对话。
- 第六十二回：宝玉、芳官、小燕饭菜剩余。
- 第七十四回：抄检大观园后司棋、晴雯、芳官、四儿被撵。

因此第一条视频的交叉验证不能只查第六十一回。

## 已确认规则

- 前八十回表示荣国府公家钱物/供应体系的常用词是 `官中`。
- `公中` 不应作为前八十回此类语境的默认候选。
- `宫中` 指宫廷或宫殿空间，不可与 `官中` 混用。
- `分例` 是基础版本中多处出现的用词，涉及供给份额时优先核对原文。
- 生僻字/异体字图片保留为字形证据，未确认前不替换。

## 三文件一致性

三份输出的文本内容应满足：

- `.txt` 的非空行列表；
- `.srt` 每个字幕块的正文列表；
- `.transcript.json` 中 `segments[].content` 列表；

三者数量、顺序和文本完全一致。第一条视频当前校验结果为 371 条一致文本。
