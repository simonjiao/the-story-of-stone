# 红楼梦交互机器人愿景

本项目的长期目标是构建一个可修订、可追溯、可自定义风格的《红楼梦》交互机器人。它不只是回答“红楼梦里发生了什么”，还要能围绕不同版本、不同研究资料、不同讲解风格，与用户持续讨论和校订。

## 用户体验目标

- 用户可以选择自己偏好的对话风格。
- 用户可以指定讨论维度，例如文本细读、人物关系、版本差异、脂批、制度器物、叙事结构、研究观点。
- 用户可以追问证据，机器人应返回原文、注释、批语、研究资料或讲解语料出处。
- 用户可以指出错误，系统能记录校订结论，并在后续对话中复用。

## 资料分类

- 基础资料：可直接引用和校订的文本证据。当前《红楼梦》原文、注释/校记、图片字形和元数据属于第一批基础资料。
- 扩展基础资料：后续新增的其他《红楼梦》发行版本，也属于可直接引用资料，用于版本比较和异文校验。
- 研究资料：论文、专著摘录、讲义、札记等，用于提供观点和研究脉络。引用时需要标明来源，不覆盖基础资料。
- 风格资料：视频转录、讲稿、解读文章等，用于生成不同对话风格和讲解路径。风格资料附加在基础资料之上，不能作为原文校订的最高证据。
- 校订记录：用户或 agent 确认过的术语、原文引用、异文、风格约束，是后续对话和校订的可复用结论。

## 风格体系

风格不是原文证据。风格用于控制表达方式、讲解习惯、分析路径和语气，不能覆盖原文校订结论。

当前已确定的风格：

- `不红居士`: 来源于 B 站“红楼梦文本探究”视频转录，偏细读、拆解细节、从人物和制度处追问动机。

注意：`不红居士` 是项目中给这组讲解语料定义的风格名；转录文本里讲解者自称如 `不红君`，仍应按原转录内容保留，不做风格名替换。

## 对话时的证据优先级

1. 原文、回目、注释、批语和图片字形证据。
2. 已确认的校订记录。
3. 版本差异和研究资料。
4. 讲解风格语料。
5. 模型自身推理。

机器人可以用风格语料组织回答，但关键断言必须能追溯到更高优先级证据。

## 后续能力

- 支持用户选择一个或多个风格档案。
- 支持在对话中切换“原文证据优先”“研究观点优先”“讲解风格优先”等模式。
- 支持比较多个发行版本的异文。
- 支持将用户反馈写入校订记录。
- 支持多 agent 使用同一知识库和同一套校订记录。
