# ISSUE-006: 长回答生成中的停止控制缺少可访问标签

## 状态

OPEN

## 等级

P3

## 关联测试

| 用例 ID | 状态 | 说明 |
| --- | --- | --- |
| `LONG-STOP-20260508` | PASS | 长回答生成可中断，停止后页面显示 `继续生成`。 |

## 影响

长回答生成中的停止控制功能可用，但在可访问 DOM 中表现为无文本、无 `aria-label` 的泛用按钮。视觉用户可以通过图标理解，但自动化测试和辅助技术难以稳定识别该按钮的语义。

该问题不阻断核心聊天能力，也不影响停止功能本身，但会降低可访问性和测试稳定性。

## 证据

- 生成 1000 行清单时，测试立即点击底部无标签生成控制按钮。
- 生成停止在第 6 行附近，未出现 `第 1000 行`。
- 停止后页面出现 `继续生成`，说明中断功能生效。
- 生成中控制按钮在可访问 DOM 中没有明确 `停止生成` 或 `Stop generating` 标签。

## 当前判断

这是前端可访问性问题。当前可以通过视觉图标使用，不需要后端或 P0 控制面修复。

## 后续动作

建议在 Open WebUI 前端为生成中的停止按钮补充稳定可访问名称，例如：

```text
aria-label="停止生成"
```

修复后复测 `LONG-STOP-20260508`，确认自动化和辅助技术都能通过语义标签定位停止控制。
