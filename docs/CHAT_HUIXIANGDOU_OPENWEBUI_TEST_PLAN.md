# chat.huixiangdou.top Open WebUI 交互测试计划

本文档用于按真实浏览器交互方式测试 `https://chat.huixiangdou.top`。测试参考 Open WebUI 常见能力面：本地邮箱/密码登录、模型与权限、聊天输入与流式响应、会话历史与搜索、文件上传/RAG、Knowledge、设置与数据控制。

## 测试原则

- 使用浏览器交互方式执行，不直接调用后端接口替代用户路径。
- 使用专用测试账号：`simon.jiao1@icloud.com`。
- 密码只在测试运行时输入，不写入仓库、截图、日志或测试产物。
- 已允许创建真实后端数据；测试数据统一使用明显前缀 `codex-test-YYYYMMDD-HHMM`。
- 除测试自己创建的会话、文件、Knowledge 外，不修改或删除已有用户数据。
- 每个用例记录：时间、浏览器、当前 URL、操作步骤、预期结果、实际结果、截图或 DOM 证据、失败复现步骤。

## 通过标准

- 认证路径稳定：可登录、会话可保持、退出后受保护页面不可访问。
- 聊天路径稳定：可选择可用模型，消息可发送，响应可完整显示，流式输出可结束或停止。
- 上下文正确：多轮追问能引用前文约束，刷新或重开会话后历史仍可恢复。
- 数据控制清晰：会话、文件、搜索、重命名、删除或归档等操作有明确结果和可恢复路径。
- RAG/文件路径可用：上传成功后，模型能基于文件内容回答，不能访问时给出清晰错误。
- 错误可解释：空输入、超长输入、网络中断、无权限功能等场景不应造成页面卡死或静默失败。
- UI 可用性达标：关键按钮可见、状态可感知、输入焦点和快捷键符合常规 Web App 预期。

问题等级：

- P0：阻断登录、核心聊天不可用、数据泄漏或明显越权。
- P1：核心路径失败，如消息无法发送、会话无法保存、文件 RAG 不可用。
- P2：重要但有绕过方式的问题，如搜索不稳定、按钮状态错误、部分渲染异常。
- P3：文案、视觉、轻微可用性问题。
