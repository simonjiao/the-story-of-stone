# 项目概览

本项目的长期目标是构建一个可修订、可追溯、可自定义风格的《红楼梦》交互机器人。当前阶段先把《红楼梦》相关视频转录和基础版本文本放进同一套可校验工作流中，支持后续多 agent 协作校订。

## 项目做什么

- 下载 B 站“红楼梦文本探究”合集视频。
- 从视频提取音频，并生成机器转录文本、字幕和转录 JSON。
- 抽取当前《红楼梦》基础版本文本、注释/校记、目录、图片字形和元数据。
- 对转录文本中的原文引用、脂批、章回名、术语和同音词做交叉验证。
- 规划一个跨机器可访问的校验知识库服务，给多个 agent 共用同一套原文证据、研究资料、风格语料和校订记录。
- 为后续交互机器人准备可选择的对话风格和讨论维度。

## 资料分类

- 基础资料：当前《红楼梦》原文、注释/校记、图片字形和元数据，可直接引用和用于校订。
- 扩展基础资料：后续新增的其他《红楼梦》发行版本，同样可直接引用，用于版本比较。
- 研究资料：后续新增的论文、专著摘录、讲义、札记等，用于观点和研究脉络。
- 风格资料：视频转录、讲稿、解读文章等，只影响表达方式、讲解习惯和分析路径。
- 校订记录：用户或 agent 确认过的修改、证据和判断。

## 主要输入

- `books/`: 原始书籍文件目录。
- `https://space.bilibili.com/558777092/lists`: B 站空间合集来源。
- `resources/hongloumeng_asr_glossary.txt`: ASR 术语词表。

## 主要输出

- `resources/styles/buhongjushi/transcripts/`: 每条视频的 `.txt`、`.srt`、`.transcript.json`。
- `resources/styles/buhongjushi/metadata/`: 视频合集清单和运行状态。
- `resources/base/hongloumeng/chapters_txt/`: 按回整理的基础版本正文和注释。
- `resources/base/hongloumeng/sections_txt/`: 基础版本全部分节文本。
- `resources/base/hongloumeng/images/`: 基础版本中的图片字形资源。

## 风格资料

B 站“红楼梦文本探究”视频转录是当前第一批讲解风格语料，项目内命名为 `不红居士` 风格。

这个风格名用于后续交互机器人的表达方式和讲解习惯配置，不替代转录原文中的讲解者自称。转录文本里的 `不红君` 仍按原内容保留。风格资料附加在基础资料之上，不能覆盖原文、注释、批语、图片字形或校订记录。

## 核心原则

- 转录文本是机器初稿，不能直接当作可引用文本。
- 原文引用、脂批和章回名必须回到基础版本文本确认。
- 一个视频可能跨多个章节，校验时不能只查视频标题对应章节。
- 向量召回只负责快速找到可能相关段落，最终校订必须落到原文证据。
- 字形图片是证据资源，未确认前不强行替换成某个 Unicode 字。
- 风格语料只能影响表达方式，不能覆盖原文证据和校订规则。
